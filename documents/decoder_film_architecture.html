2

Decoder에서 Condition을 주입하는 대안적 방법들

FiLM

현재 사용
γ, β = Linear(c) # (B, C_out)
h_out = γ ⊙ h + β
조건 벡터 c에서 채널별 scale(γ)과 shift(β)를 생성하여 feature map을 변조합니다. 각 채널이 독립적으로 조절되며, 같은 spatial 위치에 균일하게 적용됩니다. 가장 널리 검증된 방식입니다.
✓ 파라미터 적음 ✓ 학습 안정적 ✓ 검증된 방식 ✗ spatial 정보 없음

AdaIN

StyleGAN 계열
γ, β = Linear(c)
h_norm = (h − μ(h)) / σ(h) # instance norm
h_out = γ ⊙ h_norm + β
FiLM과 구조가 유사하지만, γ·h + β를 적용하기 전에 feature map을 Instance Normalization합니다. 이를 통해 기존 feature의 통계(평균/분산)를 완전히 제거하고, c가 새로운 통계를 주입합니다. "스타일을 완전히 교체"하는 효과가 있어 StyleGAN에서 사용됩니다.
✓ 스타일 제어 강력 ✓ FiLM 대비 normalize 추가 ✗ 정보 손실 (기존 통계 제거) ≈ FiLM과 파라미터 유사

SPADE

spatial 조건
# c가 벡터가 아닌 spatial map일 때
γ(x,y), β(x,y) = Conv(c_map)
h_out(x,y) = γ(x,y) ⊙ h_norm(x,y) + β(x,y)
FiLM/AdaIN의 spatial 확장입니다. 조건이 벡터가 아닌 2D 맵(예: segmentation map)일 때 사용합니다. pixel 위치마다 다른 γ, β를 적용하므로, "이 위치가 도로인지 건물인지"에 따라 다른 변조가 가능합니다. PCB 시스템에서는 Design Image 자체를 spatial condition으로 쓸 수 있습니다.
✓ 위치별 조건 반영 ✓ semantic layout에 강력 ✗ 파라미터 많음 (Conv 사용) ✗ 조건이 spatial map이어야 함

Cross-Attention

Diffusion 계열
Q = proj(h_flat) # feature map → tokens
K, V = proj(c)
h_out = reshape(MHA(Q, K, V))
Feature map의 각 spatial 위치를 token으로 보고, 조건 c에 대해 attention을 수행합니다. Stable Diffusion에서 text prompt를 이미지 생성에 반영할 때 이 방식을 사용합니다. 각 pixel이 c의 어떤 측면에 attend할지 선택적으로 결정합니다.
✓ 가장 유연한 표현력 ✓ 위치별 선택적 반영 ✗ O(HW) 복잡도 ✗ 파라미터/메모리 최대

현재 시스템에 대한 권장사항
PCB Elevation 생성에서 조건 c는 64차원 global vector이므로, FiLM이 가장 효율적입니다. 만약 Design Image를 spatial condition으로 직접 활용하고 싶다면 SPADE도 실험해볼 가치가 있습니다 — 이 경우 Design Image를 각 decoder 블록의 해상도에 맞게 downsample하여 입력합니다.
