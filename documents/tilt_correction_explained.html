Why here (after interpolation, before smoothing)?

After interpolation — The surface is NaN-free, so all four corner patches have valid elevation data for stable plane fitting.

Before smoothing — Gaussian smoothing preserves the true warpage extremes rather than tilt-inflated ones. If we smoothed first, the smoother would blend real warpage with the tilt artifact.

2. Why Does Tilt Exist?

Fixture Misalignment

The measurement jig may not be perfectly level, adding a systematic linear slope to every scan.

Component Weight

Heavy components near edges can cause one side to sag, introducing asymmetric tilt.

Gravitational Sag

A board supported at two edges will bow under its own weight, creating a linear gradient.

Thermal Gradients

Uneven heating during reflow can produce a temperature-driven tilt component.

This tilt is not part of the actual warpage pattern we want to study. It must be removed so the model learns only genuine deformation.

3. Algorithm — Step by Step

Step A — Extract Corner Patches

A square patch of size patch_size × patch_size (default 16×16 pixels) is taken at each of the four corners. Using a patch rather than a single pixel makes the corner elevation estimate robust to local noise.

Step B — Fit a Least-Squares Plane

We fit a plane z = a·row + b·col + c through the four (centroid, mean_z) points. With 4 data points and 3 unknowns (a, b, c), the system is overdetermined by 1 degree — the least-squares solution finds the best-fit plane that minimises the sum of squared residuals.

Solved via np.linalg.lstsq(A, z), which returns coefficients [a, b, c].

Step C — Build Full Plane & Subtract

plane(row, col) = a · row + b · col + c

flattened = data − plane
Surface − Tilt Plane = Flattened Surface

Step D — Zero-Baseline Shift

After subtracting the plane, the values may have a negative offset. We shift the entire surface so that the minimum becomes exactly 0.0:

flattened = flattened − min(flattened)

∴  min(result) = 0.0 ,   max(result) = max − min (the actual warpage range)

5. Plane Amplitude Diagnostic

The function returns plane_amplitude = max(plane) − min(plane), which tells you how much tilt was removed.

plane_amplitudeInterpretation
≈ 0.0Sample was already level — almost no correction needed
0.01 – 0.05Mild tilt — typical for well-fixtured boards
0.05 – 0.20Moderate tilt — common in production data
> 0.20Significant tilt — may indicate fixture issues or heavy sag

6. Why Zero-Baseline? — Four Reasons

① Relative Warpage Matters

PCB warpage analysis studies the shape of deformation (peaks vs valleys). The absolute elevation from the fixture is physically meaningless and varies per scan setup.

② Consistent Cross-Sample Comparison

Without zeroing, each sample has an arbitrary offset depending on how much tilt was removed. A common baseline enables fair min/max statistics across all samples.

③ Clean Input for Global Scaling

Global min/max image scaling (Step 7) benefits from global_min ≥ 0 always. Per-file range = max − 0 = max, simplifying interpretation.

④ Model Training Stability

The generative model (CVAE/DDPM) receives data globally scaled to [0, 1]. A consistent [0, max_warpage] per sample means cleaner training signal — no arbitrary offsets to learn.

7. Code Reference

def flatten_tilt(data: np.ndarray, patch_size: int = 16) -> tuple:
    H, W = data.shape
    ps = min(patch_size, H // 4, W // 4)  # clamp to avoid overlap

    # Step A: Corner patches → mean elevation + centroid
    corners = [
        (data[:ps, :ps],           ps/2,     ps/2),       # top-left
        (data[:ps, W-ps:],         ps/2,     W - ps/2),   # top-right
        (data[H-ps:, :ps],         H - ps/2, ps/2),       # bottom-left
        (data[H-ps:, W-ps:],       H - ps/2, W - ps/2),  # bottom-right
    ]

    # Step B: Build 4×3 system and solve for [a, b, c]
    A = np.empty((4, 3), dtype=np.float64)
    z = np.empty(4, dtype=np.float64)
    for i, (patch, r_center, c_center) in enumerate(corners):
        A[i] = [r_center, c_center, 1.0]
        z[i] = float(np.mean(patch))

    coeffs, _, _, _ = np.linalg.lstsq(A, z, rcond=None)

    # Step C: Build full plane and subtract
    rows, cols = np.indices((H, W), dtype=np.float64)
    plane = coeffs[0]*rows + coeffs[1]*cols + coeffs[2]

    plane_amplitude = float(plane.max() - plane.min())

    flattened = data.astype(np.float64) - plane

    # Step D: Zero-baseline shift
    flattened -= flattened.min()            # min → 0.0

    return flattened.astype(np.float32), plane_amplitude
