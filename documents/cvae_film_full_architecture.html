Conditional CVAE + FiLM

PCB Elevation Generation — Complete Architecture & Detailed Flow Guide (전체 아키텍처 상세 도식 가이드)

WHAT IS THIS SYSTEM?

This system predicts PCB warpage elevation maps from a circuit board design image. Warpage (bowing or twisting of the board) depends on manufacturing tolerances and material variation, so a single design does not produce one fixed warpage pattern — it produces a distribution of possible patterns.

A Conditional VAE (CVAE) with FiLM conditioning learns this distribution. Given a new PCB design, the model generates K different plausible elevation maps in one pass. Their mean reveals the expected warpage; their spread shows which areas are physically uncertain.

c — Condition Vector
64-dim summary of the PCB design. Computed once; fixed across all K samples. Tells the model what board this is.
z₁ — Latent Vector
64-dim random code sampled from 𝒩(0,I). Re-sampled for each of the K outputs — the sole source of diversity.
z_fused — Fused Latent
z₁ modulated by c via FiLM. Carries both "what kind of warpage" and "which design" — the decoder's starting point.
x̂ — Generated Elevation
256×256 grayscale output. Each pixel value = normalised warpage height [0,1] at that board location.
TRAINING PHASE (Steps 1–5)
Both the PCB design and the ground-truth elevation image are fed in.
The model learns to encode, fuse, and reconstruct elevation accurately.
The Elevation Encoder teaches the latent space what different warpage patterns "feel like."
INFERENCE PHASE (Step I)
Only the PCB design is needed — no elevation image required.
z₁ is sampled randomly from 𝒩(0,I); the Elevation Encoder is not used.
Run K times → K diverse, plausible elevation maps for that design.
Training Phase
1

Design Encoder — Building the Condition Vector c

deterministic · same input always gives the same output (같은 입력 → 항상 같은 출력)

The Design Encoder receives a 256×256 grayscale PCB design image plus 24 handcrafted geometric features, and compresses the design's structural properties into a single 64-dimensional vector c. Think of c as the PCB's "fingerprint" — it captures pattern density, symmetry, component layout, and wiring complexity in 64 numbers.

This vector c is then reused at 5 different points in the pipeline (once in the Latent Fusion step, and once at each of the 4 Decoder blocks) to continuously remind the model which design is being generated.

[PCB 설계 이미지와 24개의 handcrafted features를 받아서 설계의 특성을 64차원 벡터 c로 압축합니다. c는 이후 총 5곳에서 사용됩니다.]

What c encodes: The overall structure of the PCB — pattern density, symmetry, wiring complexity, component layout, edge orientation, etc. — compressed to 64 numbers. Why CNN branch + Handcrafted branch?
The CNN branch discovers spatial patterns automatically — texture gradients, local trace density, coarse spatial layout. It uses a 4×4 spatial pool (instead of a single global average) so two designs with similar overall statistics but different spatial arrangements still get different c vectors.

The Handcrafted branch injects 24 domain-knowledge features (symmetry scores, edge-orientation ratios, component density, Hu moments) that are hard to learn reliably from the small set of training designs. The two branches are concatenated and fused via a final MLP.
2

Elevation Encoder — Producing the Latent Vector z₁

stochastic · used during training only · discarded at inference (학습 시에만 사용 · 추론 시 제거)

During training, the Elevation Encoder receives the real (ground-truth) elevation image and encodes its warpage characteristics as a probability distribution — described by mean μ and log-variance log σ², each 64-dimensional.

Rather than a fixed code, a distribution is learned because one design can produce many different warpage patterns. A single sample z₁ is then drawn from this distribution using the reparameterization trick (see equation below), allowing gradients to flow back through the sampling step during training.

[실제 Elevation 이미지(GT)를 받아서 변형 특성을 확률 분포(μ, σ)로 인코딩하고, reparameterization trick으로 z₁을 샘플링합니다.]

# Reparameterization Trick
σ = exp(0.5 × log σ²)
ε ~ 𝒩(0, I) ← 표준 정규분포에서 랜덤 샘플링

z₁ — Latent Vector
shape: (B, 64) · stochastic
What z₁ captures: The "warpage personality" of a specific elevation image — how much it deviates from the average, which regions bow up vs. down, and the magnitude of the deformation. μ = "the central character of this warpage", σ = "how uncertain or variable this warpage mode is".

Why stochastic? A single PCB design produces many different warpage outcomes depending on manufacturing conditions. Modeling z₁ as a distribution (not a fixed point) lets the model represent this one-to-many mapping naturally.

Why does inference work without this encoder? The KL loss (Step 5) forces the learned z₁ distribution q(z₁|x) to stay close to 𝒩(0, I) during training. So at inference, sampling z₁ ~ 𝒩(0, I) directly yields valid warpage codes — the encoder is no longer needed.

[z₁: 변형의 특성. 왜 확률적인가: 하나의 설계에 여러 elevation 존재 가능. 추론 시: KL loss가 분포를 𝒩(0,I)에 가깝게 만들어 직접 샘플링 가능.]
Reparameterization Trick — Why it matters:
Normally you cannot backpropagate through a random sampling operation. The trick rewrites the sample as z₁ = μ + ε × σ where ε ~ 𝒩(0, I) is treated as a fixed external noise input (not a parameter). Now μ and σ are deterministic functions, and gradients flow through them normally. The randomness is "moved outside" the computational graph.
3

Latent Space Fusion — Combining z₁ and c with FiLM

FiLM application ① · 1D vector × 1D vector (1D 벡터 × 1D 벡터)

The condition vector c (design identity) modulates the latent vector z₁ (warpage sample) element-wise via FiLM (Feature-wise Linear Modulation): for each of the 64 dimensions, c decides how much to amplify or suppress that dimension of z₁ (via γ — scale), and what baseline offset to apply regardless of z₁'s value (via β — shift).

Analogy: Think of z₁ as a raw audio recording and γ/β as an equalizer whose frequency settings are determined by the design. The same raw recording sounds different depending on the EQ preset chosen by c.

[c(설계 정보)가 z₁(변형 정보)을 element-wise로 변조합니다. gamma로 스케일링, beta로 기본값을 설정합니다.]

# element-wise 적용
z_fused[0] = 1.80 × 0.50 + (-0.20) = 0.70 ← γ=1.8: 증폭됨
z_fused[1] = 0.05 × (-1.20) + 0.50 = 0.44 ← γ≈0: 거의 억제
z_fused[2] = 2.30 × 0.80 + 0.00 = 1.84 ← γ=2.3: 강하게 증폭
z_fused[3] = 0.90 × 2.00 + 1.00 = 2.80 ← γ≈1: 유지 + offset
Role of γ (scale): Amplifies (γ > 1), preserves (γ ≈ 1), or suppresses (γ ≈ 0) each dimension of z₁. Answers: "For this design, which axes of the latent space carry meaningful warpage variation?"
[설계에 따라 z₁의 각 차원을 증폭/유지/억제합니다.]

Role of β (shift): Design-specific baseline offset added after scaling. Even when z₁ = 0 (the "average" warpage), β alone injects the design's characteristic bias into the fused vector.
[설계 고유의 기본값(offset). z₁=0이어도 β만으로 설계 특성이 반영됩니다.]

Key insight: γ and β are produced by mixing all 64 dimensions of c through a learned linear layer (matrix multiply). A different design → different c → completely different γ and β → the same z₁ produces a completely different z_fused for each design.
[다른 설계 → 다른 c → 다른 γ,β → 같은 z₁이라도 설계에 따라 전혀 다른 z_fused 생성.]

Why FiLM instead of simple concatenation? Concatenating c to z₁ just appends the two — the network must learn on its own how to mix them, which is harder with limited data. FiLM gives the model a structured mechanism: c explicitly controls the interpretation of z₁ by rescaling and shifting each dimension, which is more parameter-efficient and easier to optimise.

4

FiLM-Conditioned Decoder — Reconstructing the Elevation Image

FiLM applications ②③④⑤ · 1D condition vector modulates 3D feature maps (1D 벡터가 3D feature map을 변조)

The decoder "paints" an elevation image progressively: starting from a tiny 8×8 canvas, it doubles the resolution at each of 4 upsampling blocks until reaching the full 256×256 output. At every block, the condition vector c injects design-specific guidance via FiLM — ensuring the growing image reflects the correct PCB structure at every spatial scale.

Why apply FiLM at every block, not just once? Without repeated conditioning, the design information injected via z_fused can "wash out" as it passes through multiple convolutions. Re-injecting c at each resolution keeps the design constraints active throughout the entire decoding process.

[z_fused를 8×8 feature map으로 확장한 뒤 4번의 upsample 블록으로 256×256 이미지를 생성합니다. 매 블록마다 c가 FiLM으로 주입됩니다.]

What each block is responsible for:

Block 1 (8²→16²): Board-level coarse structure — which quadrants are high/low overall
Block 2 (16²→32²): Mid-scale component group patterns — height variation at cluster level
Block 3 (32²→64²): Individual component-level detail
Block 4 (64²→128²): Fine surface texture, edge sharpness, micro-warpage
(Final Upsample + Conv, no FiLM): 128²→256² — pixel-level output refinement)

The same c feeds into all 4 FiLM blocks, but each block has independent Linear layers for γ and β. So the same design condition is "re-read" differently at each scale, producing scale-appropriate modulation (coarse layout guidance early; fine texture guidance late).
[Block1(큰 구조), Block2(중간 패턴), Block3(컴포넌트), Block4(미세 텍스처). 동일한 c가 4개 블록에 입력되지만 각 블록의 Linear는 독립적인 파라미터를 가집니다.]
Inside each Decoder block (e.g. Block 2):
1. Bilinear Upsample ×2 — doubles spatial resolution smoothly. Preferred over transposed convolution because it avoids "checkerboard" artifacts.
2. Conv2d (3×3, stride 1) — refines features after upsampling; also reduces channel count (e.g. 128→64 in Block 2).
3. BatchNorm + ReLU — BatchNorm normalises activations within each channel across the batch, stabilising training and allowing higher learning rates. ReLU discards negative values, introducing non-linearity so the model can learn complex mappings.
4. FiLM — applies design conditioning at this resolution before passing to the next block.

Loss Calculation and Training (Loss 계산 및 학습)

MSE + KL divergence · β annealing
Loss 구성
x̂ (생성된 elevation)
(B, 1, 256, 256)
vs
x (GT elevation)
(B, 1, 256, 256)
ℒ = MSE(x̂, x) + β × KL( q(z₁|x) ‖ 𝒩(0, I) )
MSE (Reconstruction Loss):
Minimises the pixel-wise squared difference between generated x̂ and the real elevation x. This signal propagates back through the entire pipeline — Decoder Conv weights, FiLM γ/β Linear layers, Latent Fusion, and the Design Encoder — updating all parameters simultaneously.
[생성된 x̂와 실제 x의 픽셀별 차이를 줄입니다. Design Encoder, Latent Fusion, Decoder 전체의 파라미터를 업데이트합니다.]
KL Divergence (Regularisation):
Pushes the Elevation Encoder's learned posterior q(z₁|x) toward the standard normal prior 𝒩(0, I). This is what makes inference possible: because z₁ always resembles 𝒩(0, I) after training, we can sample directly from 𝒩(0, I) at inference time.

Why β annealing (0 → 0.5, cyclical)? If the KL weight β is too large from the start, the model can simply collapse all z₁ values to 𝒩(0, I) trivially and learn nothing useful from the latent space (called posterior collapse). Starting β at 0 lets the model first learn to reconstruct well, then gradually introduces the KL constraint to organise the latent space — preventing collapse while maintaining diversity.
[q(z₁|x)를 𝒩(0,I)에 가깝게 만들어 추론 시 직접 샘플링 가능. β cyclical annealing(0→0.5)으로 posterior collapse를 방지합니다.]
Backpropagation Path:
Loss → Decoder (Conv weights + FiLM γ/β Linear layers) → z_fused → Latent Fusion (γ/β Linear) → two branches: (1) Elevation Encoder μ and σ heads   (2) Design Encoder CNN and MLP

All parameters are trained simultaneously end-to-end. The FiLM Linear layers learn "which γ and β values produce accurate reconstructions" — in other words, they learn how to re-interpret the design condition c to guide generation at each location in the pipeline. The reparameterization trick is what enables gradients to flow through z₁ back into the Elevation Encoder.
[Loss → Decoder → z_fused → Latent Fusion → (1) Elevation Encoder μ,σ / (2) Design Encoder. 모든 파라미터가 end-to-end로 동시 학습됩니다.]
Inference Phase
I

Inference: Generating K Elevation Maps from a New Design

Elevation Encoder removed · z₁ sampled directly from 𝒩(0,I) (Elevation Encoder 제거 · z₁은 N(0,I)에서 샘플링)

At inference time, the Elevation Encoder is discarded — we have no real elevation to encode. Instead, z₁ is sampled directly from the prior 𝒩(0, I), scaled by temperature τ. The KL regularisation from training ensures these samples are valid inputs for the decoder.

Repeating this K times for the same design (c is computed once and reused) produces K different but all plausible elevation maps, capturing the full range of warpage outcomes the model has learned for that PCB design.

[학습된 모델에서 Elevation Encoder를 제거하고 z₁을 prior에서 직접 샘플링합니다. K번 반복으로 K개의 다른 elevation을 생성합니다.]

The sole source of diversity is the random z₁ sampling. Temperature τ controls how wide that distribution is:

τ = 0.5 → z₁ values cluster near 0 → K outputs are similar (conservative, lower diversity)
τ = 1.0 → standard normal → same spread as during training (recommended starting point)
τ = 1.5 → wider spread → more diverse but quality may degrade (unseen latent regions)

Analysing the K samples: Compute pixel-wise statistics over all K elevation maps:
  • Mean map — expected warpage pattern for this design
  • Std map — uncertainty map; high std = the model predicts variable warpage here, indicating that region is sensitive to manufacturing variation
  • Low-std regions are structurally determined by the design geometry; high-std regions reflect physical uncertainty

[다양성의 유일한 원천은 z₁의 랜덤 샘플링. τ로 다양성 제어: τ=0.5(안전), τ=1.0(표준), τ=1.5(다양하지만 품질 저하 가능). 50개 샘플 → mean(평균 elevation) + std(불확실성 맵).]
