Conditional CVAE + FiLM

PCB Elevation Map 생성 — 전체 아키텍처 상세 도식 가이드

이 시스템은 무엇인가?

이 시스템은 회로 기판 설계 이미지로부터 PCB 휨(warpage) elevation map을 예측합니다. PCB의 휨 현상(보드가 구부러지거나 뒤틀리는 현상)은 제조 공차와 재료 편차에 따라 달라지므로, 동일한 설계라 하더라도 하나의 고정된 휨 패턴이 아닌 가능한 패턴의 분포(distribution)를 생성합니다.

FiLM 조건화를 적용한 Conditional VAE (CVAE)가 이 분포를 학습합니다. 새로운 PCB 설계가 주어지면, 모델은 한 번의 실행으로 K개의 서로 다른 그럴듯한 elevation map을 생성합니다. 이들의 평균은 기대되는 휨을 나타내고, 분산은 물리적으로 불확실한 영역을 보여줍니다.

c — Condition Vector
PCB 설계를 요약한 64차원 벡터. 한 번만 계산되며 K개 샘플 전체에 고정됩니다. 모델에게 어떤 보드인지를 알려줍니다.
z₁ — Latent Vector
𝒩(0,I)에서 샘플링된 64차원 랜덤 코드. K개 출력 각각에 대해 재샘플링되며 — 다양성의 유일한 원천입니다.
z_fused — Fused Latent
FiLM을 통해 c로 변조된 z₁. "어떤 종류의 휨인지"와 "어떤 설계인지" 모두를 담고 있으며 — 디코더의 출발점입니다.
x̂ — 생성된 Elevation
256×256 grayscale 출력. 각 픽셀값 = 해당 보드 위치에서의 정규화된 휨 높이 [0,1].
학습 단계 (단계 1–5)
PCB 설계 와 실제(ground-truth) elevation 이미지가 모두 입력됩니다.
모델은 인코딩, 융합, elevation 재구성을 정확하게 학습합니다.
Elevation Encoder가 잠재 공간에 다양한 휨 패턴의 "느낌"을 가르칩니다.
추론 단계 (단계 I)
PCB 설계만 필요합니다 — elevation 이미지 불필요.
z₁은 𝒩(0,I)에서 직접 랜덤 샘플링되며, Elevation Encoder는 사용되지 않습니다.
K번 실행 → 해당 설계에 대한 K개의 다양하고 그럴듯한 elevation map 생성.

1

Design Encoder — Condition Vector c 생성

결정론적 · 동일 입력은 항상 동일 출력

Design Encoder는 256×256 grayscale PCB 설계 이미지와 24개의 handcrafted 기하학적 특성을 입력받아, 설계의 구조적 속성을 단일 64차원 벡터 c로 압축합니다. c를 PCB의 "지문"이라고 생각하면 됩니다 — 64개의 숫자로 패턴 밀도, 대칭성, 컴포넌트 배치, 배선 복잡도를 담습니다.

이 벡터 c는 이후 파이프라인의 5개 지점에서 재사용됩니다 (Latent Fusion에서 한 번, 4개의 Decoder 블록 각각에서 한 번씩) — 생성 중인 설계가 어떤 것인지 모델이 계속 인식할 수 있도록 합니다.

CNN 브랜치 + Handcrafted 브랜치를 함께 사용하는 이유?
CNN 브랜치는 공간 패턴을 자동으로 발견합니다 — 텍스처 그래디언트, 지역적 트레이스 밀도, 대략적인 공간 배치. 단일 전역 평균 대신 4×4 공간 풀링을 사용하므로 전체 통계는 비슷하지만 공간 배치가 다른 두 설계도 서로 다른 c 벡터를 갖습니다.

Handcrafted 브랜치는 소수의 학습 설계만으로는 안정적으로 학습하기 어려운 24개의 도메인 지식 특성(대칭성 점수, 에지 방향 비율, 컴포넌트 밀도, Hu 모멘트)을 주입합니다. 두 브랜치는 concatenation된 후 최종 MLP로 융합됩니다.

Elevation Encoder — Latent Vector z₁ 생성

확률적 · 학습 시에만 사용 · 추론 시 제거

학습 중, Elevation Encoder는 실제(ground-truth) elevation 이미지를 받아 그 휨 특성을 확률 분포로 인코딩합니다 — 평균 μ와 로그 분산 log σ²으로 표현되며, 각각 64차원입니다.

하나의 설계가 다양한 휨 패턴을 생성할 수 있으므로, 고정된 코드 대신 분포를 학습합니다. reparameterization trick(아래 수식 참조)을 사용하여 이 분포에서 z₁을 샘플링하면, 학습 중 샘플링 단계를 통해 gradient가 역전파될 수 있습니다.

# Reparameterization Trick
σ = exp(0.5 × log σ²)
ε ~ 𝒩(0, I) ← 표준 정규분포에서 랜덤 샘플링
z₁ = μ + ε × σ ← gradient가 μ, σ를 통해 역전파 가능
z₁ — Latent Vector
shape: (B, 64) · 확률적
z₁이 담는 정보: 특정 elevation 이미지의 "휨 개성" — 평균으로부터 얼마나 벗어났는지, 어느 영역이 위로 또는 아래로 휘는지, 변형의 크기. μ = "이 휨의 핵심 특성", σ = "이 휨 모드가 얼마나 불확실하거나 가변적인가".

왜 확률적인가? 하나의 PCB 설계는 제조 조건에 따라 다양한 휨 결과를 만들어냅니다. z₁을 분포(고정 점이 아닌)로 모델링하면 이 일대다 매핑을 자연스럽게 표현할 수 있습니다.

이 인코더 없이 추론이 가능한 이유? 학습 중 KL loss(단계 5)가 학습된 z₁ 분포 q(z₁|x)를 𝒩(0, I)에 가깝게 유지합니다. 따라서 추론 시 z₁ ~ 𝒩(0, I)를 직접 샘플링해도 유효한 휨 코드를 얻을 수 있습니다 — 인코더가 더 이상 필요 없습니다.
Reparameterization Trick — 왜 중요한가:
일반적으로 랜덤 샘플링 연산을 통해 역전파할 수 없습니다. 이 트릭은 샘플을 z₁ = μ + ε × σ로 다시 표현합니다. 여기서 ε ~ 𝒩(0, I)는 고정된 외부 노이즈 입력으로 (파라미터가 아님) 취급됩니다. 이제 μ와 σ는 결정론적 함수가 되어 gradient가 정상적으로 흐릅니다. 무작위성이 계산 그래프의 "바깥으로 이동"합니다.

Latent Space Fusion — FiLM으로 z₁과 c 결합

FiLM 적용 ① · 1D 벡터 × 1D 벡터

condition 벡터 c(설계 정보)가 latent 벡터 z₁(휨 샘플)을 FiLM (Feature-wise Linear Modulation)을 통해 element-wise로 변조합니다: 64차원 각각에 대해, c는 z₁의 해당 차원을 얼마나 증폭 또는 억제할지(γ — 스케일)와 z₁의 값에 무관하게 어떤 기준 오프셋을 적용할지(β — 이동)를 결정합니다.

비유: z₁을 원본 오디오 녹음, γ/β를 이퀄라이저라고 생각하세요. 주파수 설정이 설계에 의해 결정됩니다. 동일한 원본 녹음이라도 c가 선택한 EQ 프리셋에 따라 다르게 들립니다.

FiLM Fusion 연산 과정
c
(B, 64)
+
z₁
(B, 64)
↓
γ = Linear(c)
W_γ(64×64) @ c + b_γ → (B, 64)

element-wise: z_fused[i] = γ[i] × z₁[i] + β[i]
# 64차원 중 앞 4개만 추적하는 예시

z₁ = [ 0.50, -1.20, 0.80, 2.00, ... ] ← 랜덤 샘플링된 latent
c = [ 0.30, 0.70, -0.50, 1.10, ... ] ← Design Encoder 출력

# c가 학습된 Linear를 통과하여 γ, β 생성
γ = [ 1.80, 0.05, 2.30, 0.90, ... ] ← 채널별 scale
β = [-0.20, 0.50, 0.00, 1.00, ... ] ← 채널별 shift

# element-wise 적용
z_fused[0] = 1.80 × 0.50 + (-0.20) = 0.70 ← γ=1.8: 증폭됨
z_fused[1] = 0.05 × (-1.20) + 0.50 = 0.44 ← γ≈0: 거의 억제
z_fused[2] = 2.30 × 0.80 + 0.00 = 1.84 ← γ=2.3: 강하게 증폭
z_fused[3] = 0.90 × 2.00 + 1.00 = 2.80 ← γ≈1: 유지 + offset
γ (스케일)의 역할: z₁의 각 차원을 증폭(γ > 1), 유지(γ ≈ 1), 또는 억제(γ ≈ 0)합니다. "이 설계에서 잠재 공간의 어떤 축이 의미 있는 휨 변동을 담고 있는가?"라는 질문에 답합니다.

β (이동)의 역할: 스케일링 후 추가되는 설계 고유의 기준 오프셋. z₁ = 0("평균" 휨)이어도, β만으로도 융합 벡터에 설계의 특성적 편향이 주입됩니다.

핵심 인사이트: γ와 β는 학습된 선형 레이어(행렬 곱)를 통해 c의 64차원 전체를 혼합하여 생성됩니다. 다른 설계 → 다른 c → 완전히 다른 γ, β → 같은 z₁이어도 설계마다 완전히 다른 z_fused를 생성합니다.

단순 concatenation 대신 FiLM을 사용하는 이유? c를 z₁에 단순히 붙이면(concatenation) 두 벡터가 나란히 추가될 뿐 — 모델이 스스로 혼합 방법을 학습해야 하므로 데이터가 적을 때 더 어렵습니다. FiLM은 모델에 구조화된 메커니즘을 제공합니다: c가 각 차원을 재조정하고 이동시켜 z₁의 해석을 명시적으로 제어합니다 — 파라미터 효율적이며 최적화하기 쉽습니다.

FiLM 조건화 Decoder — Elevation 이미지 재구성

FiLM 적용 ②③④⑤ · 1D condition 벡터가 3D feature map을 변조

디코더는 elevation 이미지를 점진적으로 "그립니다": 작은 8×8 캔버스에서 시작하여 4개의 업샘플링 블록을 거치면서 매번 해상도를 2배로 키워 최종 256×256 출력에 도달합니다. 모든 블록에서 condition 벡터 c가 FiLM을 통해 설계별 가이던스를 주입합니다 — 성장하는 이미지가 모든 공간 스케일에서 올바른 PCB 구조를 반영하도록 보장합니다.

왜 매 블록마다 FiLM을 적용하는가? 반복 조건화 없이는, z_fused를 통해 주입된 설계 정보가 여러 컨볼루션을 거치면서 "희석"될 수 있습니다. 각 해상도에서 c를 재주입함으로써 전체 디코딩 과정에 걸쳐 설계 제약이 활성 상태로 유지됩니다.

Block 1 (8²→16²): 보드 수준의 대략적인 구조 — 어떤 사분면이 전체적으로 높고 낮은지
Block 2 (16²→32²): 중간 스케일 컴포넌트 그룹 패턴 — 클러스터 수준의 높이 변동
Block 3 (32²→64²): 개별 컴포넌트 수준의 세부 사항
Block 4 (64²→128²): 미세 표면 텍스처, 에지 선명도, 미세 휨
(최종 Upsample + Conv, FiLM 없음): 128²→256² — 픽셀 수준 출력 정제)

동일한 c가 4개의 FiLM 블록 모두에 입력되지만, 각 블록은 γ와 β에 대한 독립적인 Linear 레이어를 가집니다. 따라서 동일한 설계 조건이 각 스케일에서 "다르게 읽혀" 스케일에 맞는 변조를 생성합니다 (초반에는 대략적인 레이아웃 가이던스; 후반에는 세밀한 텍스처 가이던스).
각 Decoder 블록 내부 구조 (Block 2 예시):
1. Bilinear Upsample ×2 — 공간 해상도를 부드럽게 2배로 키웁니다. 전치 컨볼루션 대신 사용하는 이유는 "체커보드" 아티팩트를 방지하기 때문입니다.
2. Conv2d (3×3, stride 1) — 업샘플링 후 특성을 정제하고 채널 수도 줄입니다(Block 2에서 128→64).
3. BatchNorm + ReLU — BatchNorm은 배치 내 각 채널의 활성화를 정규화하여 학습을 안정화하고 더 높은 학습률을 허용합니다. ReLU는 음수 값을 제거하여 비선형성을 도입함으로써 모델이 복잡한 매핑을 학습할 수 있게 합니다.
4. FiLM — 다음 블록으로 전달하기 전 이 해상도에서 설계 조건화를 적용합니다.

MSE (재구성 손실):
생성된 x̂와 실제 elevation x 사이의 픽셀별 제곱 차이를 최소화합니다. 이 신호는 전체 파이프라인을 통해 역전파됩니다 — Decoder Conv 가중치, FiLM γ/β Linear 레이어, Latent Fusion, Design Encoder — 모든 파라미터를 동시에 업데이트합니다.
KL Divergence (정규화):
Elevation Encoder의 학습된 사후분포 q(z₁|x)를 표준 정규 사전분포 𝒩(0, I)에 가깝게 만듭니다. 이것이 추론을 가능하게 합니다: 학습 후 z₁이 항상 𝒩(0, I)와 유사하므로 추론 시 𝒩(0, I)에서 직접 샘플링할 수 있습니다.

왜 β annealing (0 → 0.5, 주기적)인가? 처음부터 KL 가중치 β가 너무 크면, 모델이 모든 z₁을 단순히 𝒩(0, I)로 붕괴시켜 잠재 공간에서 유용한 것을 학습하지 못할 수 있습니다 (posterior collapse라고 불림). β를 0에서 시작하면 모델이 먼저 재구성을 잘 학습하고, 이후 점진적으로 KL 제약이 도입되어 잠재 공간을 조직화합니다 — collapse를 방지하면서 다양성을 유지합니다.
역전파 경로:
Loss → Decoder (Conv 가중치 + FiLM γ/β Linear 레이어) → z_fused → Latent Fusion (γ/β Linear) → 두 갈래: (1) Elevation Encoder μ와 σ 헤드   (2) Design Encoder CNN과 MLP

모든 파라미터가 end-to-end로 동시에 학습됩니다. FiLM Linear 레이어는 "어떤 γ와 β 값이 정확한 재구성을 만드는가"를 학습합니다 — 다시 말해, 파이프라인의 각 위치에서 생성을 안내하기 위해 설계 조건 c를 어떻게 재해석할지를 학습합니다. Reparameterization trick이 z₁을 통해 Elevation Encoder로 gradient가 역전파될 수 있게 합니다.

FiLM: 학습 단계 vs 추론 단계, 그리고 단순 Concatenation과의 차이

FiLM의 작동 원리 심층 분석 · 설계 선택의 근거

FiLM이 단순히 "조건 벡터를 더하는 방법 중 하나"가 아니라 왜 이 구조가 선택되었는지, 학습과 추론 각 단계에서 FiLM이 실제로 무엇을 하는지, 그리고 단순한 Concatenation과 어떻게 다른지를 구체적으로 설명합니다.

A. 학습 단계에서 FiLM이 하는 일

FiLM의 Linear 레이어(γ, β를 생성하는)는 랜덤으로 초기화된 후 end-to-end로 학습됩니다. 학습 중 순전파(forward pass)에서는 다음 과정이 일어납니다:

역전파가 FiLM에서 하는 일:
MSE loss의 gradient가 FiLM 변조(γ⊙h + β)를 통해 역전파되어 γ/β를 생성하는 Linear 레이어의 가중치 W_γ, W_β로 흘러들어갑니다. 이를 통해 모델은 "어떤 채널 재조정이 각 설계에 대해 정확한 elevation 재구성을 만드는가"를 학습합니다.

Latent Fusion의 FiLM에서는 gradient가 Design Encoder와 Elevation Encoder 양쪽으로도 흘러가, c(설계 표현)와 z₁(휨 잠재 코드) 자체도 FiLM 변조가 잘 되도록 함께 조정됩니다.

결과: FiLM Linear 레이어는 "설계 A��서는 채널 5(코너 응력 감지)를 2배 증폭하고, 설계 B에서는 채널 12(중앙부 곡률)를 억제하라"는 구체적인 변조 전략을 학습합니다.
B. 추론 단계에서 FiLM이 하는 일

추론 시, 모든 파라미터(W_γ, W_β 포함)는 고정(frozen)됩니다. 동일한 설계 이미지는 항상 동일한 c를 생성하고, 고정된 W_γ, W_β를 통해 항상 동일한 γ, β를 생성합니다.

추론에서 FiLM의 역할 요약:
FiLM은 "설계별 해석 필터"로서 작동합니다. 동일한 Decoder 아키텍처와 가중치를 사용하면서도, 설계 A의 c를 주입하면 설계 A에 적합한 휨 패턴을, 설계 B의 c를 주입하면 설계 B에 적합한 패턴을 생성합니다. z₁의 무작위성이 각 설계 내의 다양성을 만들고, FiLM의 γ, β가 그 다양성이 "어떤 설계의 특성을 따르는가"를 결정합니다.
C. Decoder에서 단순 Concatenation과의 차이

만약 각 Decoder 블록에서 c를 직접 feature map에 concatenation한다면 어떻게 될까요? (예: c를 공간 크기로 tiling하여 채널 방향으로 concat) 아래에서 두 방식을 구체적으로 비교합니다.

FiLM 방식:
  γ, β = Linear(c)  ← 2×Linear 파라미터만 추가
  h_out = γ ⊙ h + β  ← 명시적 채널별 게이팅

Concat 방식:
  h_cat = cat(h, tile(c))  ← c를 공간 크기로 복제 후 채널 concat
  h_out = Conv(h_cat)  ← Conv 파라미터, 더 많고 최적화 어려움

데이터가 적을 때 FiLM이 유리한 두 가지 이유:

(1) Conv 커널의 용량 경쟁 문제:
Concatenation 방식에서는 Conv 커널이 "공간 특성 추출"과 "조건 벡터 혼합"을 동시에 수행해야 합니다. 이 두 역할이 동일한 파라미터를 두고 경쟁하므로, 데이터가 적을 때 둘 다 제대로 학습하기 어렵습니다. FiLM은 이 역할을 명확히 분리합니다: Conv는 공간 특성 추출만, Linear는 조건 변조만 담당합니다.

(2) 조건 영향의 비구조성:
Concatenation에서는 조건 벡터의 영향이 원칙 없이 모두 혼합됩니다. 반면 FiLM은 명시적인 구조를 가집니다: c → γ, β는 곱셈(게이팅)과 덧셈(오프셋)이라는 명확한 의미를 갖는 게이트입니다. 모델이 "설계 A에서는 코너 응력 채널(채널 5)을 2배 증폭"하는 것을 구조화되고 파라미터 효율적인 방식으로 학습할 수 있습니다.

파라미터 효율성: FiLM은 블록당 2×Linear(c_dim, C_out) 파라미터만 추가하는 반면, Concatenation은 더 많은 파라미터를 갖는 Conv(C_out + c_dim → C_out)를 추가하며 최적화도 더 어렵습니다.
직관적 비유:
FiLM은 각 채널에 독립적인 볼륨 조절 노브를 다는 것과 같습니다. 설계 정보가 "채널 5의 볼륨을 2배로, 채널 12는 거의 끄라"는 구체적인 지시를 내립니다.

Concatenation은 모든 신호를 하나의 케이블에 섞어 보내는 것과 같습니다. 다음 레이어가 "이 섞인 신호에서 어떤 것이 공간 패턴이고 어떤 것이 조건 정보인지" 스스로 분리해야 하므로, 데이터가 적을 때 이 분리가 잘 되지 않습니다.

β

Loss 공식 — β와 free_bits의 역할

CVAE 학습의 핵심 균형 파라미터

CVAE의 총 loss는 두 항의 합입니다. 이 두 항은 서로 상충(trade-off)하며, β와 free_bits가 이 균형을 조절합니다.

Total Loss = Recon Loss + β × KL Loss
────────────────────────────────────────────────────
Recon = MSE(x̂, x) + spectral_weight × FFT_MSE(x̂, x)
KL = −½ Σj max( free_bits, 1 + logσ²j − μj² − σ²j )
β — Prior 정렬 강도 (beta_max로 제어)

β는 KL penalty의 가중치입니다. KL Loss는 학습된 posterior q(z₁|x,c) = 𝒩(μ,σ²)가 prior p(z₁) = 𝒩(0,I)와 얼마나 다른지를 측정합니다.

β 값효과
β → 0 KL 페널티 없음. encoder가 posterior를 prior에서 자유롭게 멀어지게 학습. μ가 0에서 멀어지고 σ가 수축함. → 추론 시 z~𝒩(0,I) 샘플이 out-of-distribution
β = 1 표준 VAE. Recon과 KL이 동등한 비중. μ≈0, σ≈1로 수렴하는 강한 압력.
β > 1 β-VAE. KL이 더 강조됨. posterior가 prior에 매우 가깝지만 recon 품질 저하 가능.
Cyclical Annealing 스케줄 (현재 설정)
# config.txt 기본값 beta_max = 0.2 beta_cycles = 4 epochs = 200 # 한 사이클 = 50 epoch # epoch 0 → β = 0.00 # epoch 25 → β = 0.10 # epoch 49 → β = 0.196 # epoch 50 → β = 0.00 (리셋)
사이클마다 β를 0으로 리셋하는 이유: 초반엔 recon을 먼저 학습하고, 이후 KL regularization을 강화하는 전략. posterior collapse(KL→0)를 방지하는 효과도 있음.
beta_max=0.2 문제:
KL 당기는 힘이 약해서 μ가 0에서 멀리 drift함. 진단 결과 평균 |μ|=0.56, 최대 |μ|=3.67까지 관측됨. 권장: beta_max를 1.0으로 올리기.
free_bits — Posterior Collapse 방지 하한선

free_bits는 각 latent 차원마다 KL의 최솟값을 보장하는 하한선입니다. posterior collapse — decoder가 z₁을 완전히 무시하고 조건 c만으로 출력을 결정하는 현상 — 를 방지합니다.

# 차원 j의 KL이 free_bits 미만이면 강제로 올림
KL_j_effective = max( free_bits, KL_j_raw )
# Loss에 쓰이는 KL = Σ_j KL_j_effective
free_bits가 없으면 (=0)
일부 차원의 KL이 0으로 수렴 가능
→ posterior가 prior와 완전히 같아짐
→ decoder가 해당 차원의 z₁ 정보 무시
→ posterior collapse
free_bits=0.5이면
각 차원이 최소 0.5 nats의 KL 유지
→ encoder가 z₁에 항상 정보를 담음
→ active_kl_dims가 높게 유지됨
→ 모든 64차원이 활성 상태
β와 free_bits의 상충 관계:

두 파라미터는 서로 반대 방향으로 작용합니다.

β (beta_max) ↑  → posterior를 prior 𝒩(0,I)에 가깝게 당기는 힘
free_bits ↑    → posterior가 prior에 너무 가까워지는 걸 막는 힘

현재 beta_max=0.2, free_bits=0.5 조합의 문제:
당기는 힘(β=0.2)이 약한데 막는 힘(free_bits=0.5)이 상대적으로 강함. 결과적으로 encoder는 prior에서 멀어도 페널티가 작고, z₁에 정보는 담아야 하므로 μ를 크게 유지하는 방향으로 학습됨.

권장 조합: beta_max=1.0, free_bits=0.1
당기는 힘을 강화하고 막는 힘을 줄여 μ≈0, σ≈1 수렴을 유도.
건강한 posterior 기준
평균 |μ| < 0.2
평균 σ ∈ [0.85, 1.1]
active_kl_dims > z_dim × 0.25
총 KL ∈ [z_dim × 0.3, z_dim × 1.0] nats
문제 징후

평균 σ < 0.5 → posterior 너무 좁음
active_dims < z_dim × 0.25 → collapse
Gen Std가 fold마다 동일 → decoder가 z₁ 무시

추론 시 발생하는 Prior-Posterior 불일치

학습과 추론 사이의 z₁ 분포 차이

CVAE는 학습 시와 추론 시 z₁을 서로 다른 분포에서 샘플링합니다. β가 충분히 크지 않으면 두 분포 사이에 큰 gap이 생겨, 생성 품질이 크게 저하됩니다.

학습 vs 추론: z₁ 샘플링 경로 비교
학습 시 — Posterior Sampling
실제 elevation x → Elevation Encoder → (μ, σ)
z₁ = μ + ε·σ   (ε ~ 𝒩(0,I))

z₁의 분포 = q(z₁|x,c) = 𝒩(μ, σ²)
β가 낮으면 μ≠0, σ≠1인 채로 학습됨
추론 시 — Prior Sampling
Elevation Encoder 사용 안 함
z₁ ~ 𝒩(0, I) 에서 직접 샘플링

decoder는 학습 때 본 적 없는
out-of-distribution z₁을 받게 됨
→ 비현실적 출력, 낮은 diversity
이상적:   q(z₁|x,c) ≈ p(z₁) = 𝒩(0,I)   → KL Loss → 0
실현 방법: β를 충분히 크게 → μ→0, σ→1 수렴 압력

MMD — Maximum Mean Discrepancy
MMD (Maximum Mean Discrepancy) 이란?

두 분포 P (실제 데이터)와 Q (모델 생성)가 얼마나 다른지를 측정하는 거리 지표. 값이 0이면 두 분포가 완전히 같고, 클수록 분포 간 차이가 크다.

직관적 설명
실제 샘플 집합과 생성 샘플 집합을
커널 함수를 통해 고차원 공간으로 임베딩한 뒤,
두 집합의 평균 임베딩 벡터 간 거리를 측정.

"두 분포에서 뽑은 샘플들이
통계적으로 구별 가능한가?"에 대한 답.
수식 (개념)
MMD² =
  E[k(x, x')]
  − 2·E[k(x, ŷ)]
  + E[k(ŷ, ŷ')]

k(·,·) = RBF / Gaussian kernel
x ~ P(real),  ŷ ~ Q(generated)
항의미값이 크면
E[k(x, x')] 실제 샘플끼리의 유사도 평균 실제 분포가 응집되어 있음
E[k(x, ŷ)] 실제 ↔ 생성 샘플 간 유사도 평균 두 분포가 많이 겹침 → MMD ↓
E[k(ŷ, ŷ')] 생성 샘플끼리의 유사도 평균 생성 분포가 응집되어 있음
이 프로젝트에서의 의미

Leave-one-out 평가에서 각 폴드(held-out design)마다 MMD를 계산. 실제 elevation 샘플 분포와 모델이 해당 design으로부터 생성한 샘플 분포를 비교.

관측값해석
MMD 낮음 (<0.05) 생성 분포가 실제 분포를 잘 커버함 — 이상적 상태
MMD 높음 (≥0.15) 생성 샘플이 실제와 다른 영역에 몰려 있음. prior-posterior mismatch 또는 diversity 부족이 주 원인
MMD std 큼 일부 design에는 잘 맞고 일부에는 완전히 틀림 → 특정 design 유형에 대한 일반화 불안정
Diversity Collapse와 MMD의 관계

현재 모델의 핵심 문제: 모델이 "mean predictor"는 되었지만 "generator"가 아닌 상태.
주어진 design의 평균적인 elevation은 잘 재현하지만, z를 바꿔 샘플링해도 거의 같은 출력이 나옴.

지표현재 관측값목표값의미
gen_diversity 0.000780 real_diversity의 30~300% 생성 샘플이 실제의 12% 다양성밖에 없음
active_kl_dims mean 5.7 / 64 ≥ 15 64차원 중 ~58개가 사실상 unused
MMD 0.195 ± 0.135 < 0.10 생성 분포가 실제 분포에서 멀리 떨어짐
Diversity Collapse → MMD 상승 메커니즘:
active z 차원 부족 → 생성 샘플들이 비슷한 위치에 몰림 → E[k(ŷ,ŷ')] 매우 큼
→ 실제 분포는 넓게 퍼져 있는데 생성 분포는 한 점 근처에 집중 → MMD ↑
Diversity & MMD 개선 전략
전략방법기대 효과주의점
① beta 줄이기 beta_max = 0.3 ~ 0.4 KL penalty 완화 → 더 많은 z 차원 활성화 → diversity ↑ 너무 낮으면 prior-posterior gap 재발
② free_bits 올리기 free_bits = 0.5 ~ 1.0
64차원 × 0.5 = 최소 KL 32 nats		 각 차원에 최소 정보량 보장 → collapse 방지 너무 높으면 recon에 집중 못 함
③ z_dim 줄이기 z_dim = 16 ~ 32 작은 병목 → 각 차원이 더 의미 있게 사용됨 표현력 손실 가능성
④ Cyclical Annealing 조정 사이클 수 ↑ (4→8), 각 사이클 내 상승 비율 ↓ recon → KL 전환을 더 자주, 더 부드럽게 학습 불안정 가능성
⑤ InfoVAE / MMD-VAE KL term을 MMD term으로 교체 직접 분포 정렬 → prior-posterior gap 제거 구현 변경 필요, hyperparameter 민감
권장 우선순위:
① + ② 조합을 먼저 시도 (beta_max=0.35, free_bits=0.5).
active_kl_dims가 mean ≥ 15개로 올라오면 gen_diversity와 MMD도 자연히 개선됨.
그래도 안 되면 ③ (z_dim 축소)을 고려.

beta_cycles — Cyclical KL Annealing
beta_cycles 란?

학습 전체 epoch을 N개의 사이클로 나누고, 각 사이클마다 β를 0 → beta_max로 선형 증가시킨 뒤 다음 사이클 시작 시 다시 0으로 리셋하는 스케줄. 단순히 "beta를 서서히 올린다"보다 더 많은 latent 차원을 활성화할 수 있다.

스케줄 시각화 (200 epoch, beta_cycles=4, beta_max=0.4)
# cycle_len = 200 / 4 = 50 epoch # get_cyclical_beta(epoch, total=200, beta_max=0.4, n_cycles=4) epoch 0 → 49 : β 0.00 → 0.40 # 1번째 사이클 epoch 50 → 99 : β 0.00 → 0.40 # 2번째 사이클 (리셋) epoch 100 → 149: β 0.00 → 0.40 # 3번째 사이클 (리셋) epoch 150 → 199: β 0.00 → 0.40 # 4번째 사이클 (리셋)
사이클 초반 — β ≈ 0 구간
KL penalty 없음
→ 모델이 z에 자유롭게 정보를 담음
→ 새로운 latent 차원들이 활성화되는 기회
→ reconstruction 품질이 먼저 올라감
사이클 후반 — β → beta_max 구간
KL penalty 강해짐
→ 활성화된 z를 prior에 정렬
→ μ → 0, σ → 1 방향으로 수렴
→ prior-posterior gap 감소
왜 리셋이 도움이 되는가

단순히 β를 0에서 1까지 한 번만 올리면 (1 사이클), 한 번의 기회에 활성화된 차원들만 살아남고 나머지는 영구적으로 collapse.
사이클마다 리셋하면, 매 사이클 초반마다 새로운 차원이 활성화될 기회를 얻고, 이전 사이클에서 정렬된 차원 위에 추가로 쌓이는 형태로 학습이 진행됨.

사이클 수동작기대 효과
1 (단순 annealing) β 0 → max 한 번 초반에 활성화된 소수 차원만 살아남음
4 (현재) 50 epoch 주기로 리셋 매 사이클마다 추가 차원 활성화 기회
6 (권장) 33 epoch 주기로 리셋 더 잦은 활성화 기회 → active dims 증가
beta_cycles vs free_bits — 역할 비교

두 파라미터 모두 posterior collapse 방지를 목표로 하지만 작동 방식이 다르다.

파라미터작동 방식한계
beta_cycles 학습 스케줄 조정 — 리셋마다 새 차원이 활성화될 기회 부여 free_bits 천장이 낮으면 리셋해도 결국 같은 차원 수에 수렴
free_bits 각 차원의 최소 KL 보장 — 차원당 최소 정보량 강제 너무 높으면 reconstruction 품질 저하
현재 문제 (free_bits=0.1, z_dim=64):
공짜 KL 예산 = 64 × 0.1 = 6.4 nats
실제 active_kl_dims = 5.7 → 모델이 정확히 공짜 예산 한도까지만 사용하고 멈춤.
beta_cycles를 늘려도 free_bits 천장 자체를 뚫지는 못함. free_bits를 0.5로 올리면 예산이 32 nats로 늘어나 active dims ~30개 기대.
파라미터 조합 가이드
상황권장 설정이유

(collapse 심각)		 free_bits ↑ (0.1→0.5)
beta_max ↓ (0.8→0.4)
beta_cycles ↑ (4→6)		 free_bits가 가장 직접적. beta 낮추면 reconstruction이 z 사용을 유도. cycles 증가는 보조적
