1

Design Encoder — Condition 벡터 c 생성

deterministic · 같은 입력 → 항상 같은 출력

PCB 설계 이미지와 handcrafted features를 받아서, 이 설계의 특성을 64차원 벡터 하나로 압축합니다. 이 벡터 c는 이후 모든 단계에서 "이 설계가 어떤 설계인지"를 알려주는 역할을 합니다.

c가 담고 있는 정보: PCB 설계의 전체적인 구조 — 패턴 밀도, 대칭성, 배선 복잡도 등이 64개 숫자로 인코딩됩니다. 같은 설계 이미지를 넣으면 항상 동일한 c가 나옵니다. 이 c는 이후 Latent Fusion과 Decoder 4개 블록, 총 5곳에서 사용됩니다.
2

Elevation Encoder — Latent 벡터 z₁ 생성

stochastic · 학습 시에만 사용 · 추론 시 제거

실제 Elevation 이미지(ground truth)를 받아서, 이 elevation의 "변형 특성"을 확률 분포(μ, σ)로 인코딩합니다. 여기서 reparameterization trick으로 z₁을 샘플링합니다.

Elevation Encoder 내부 구조
Elevation Image (GT)
1ch × 256×256
↓
CNN: Conv(1→32→64→128→256)
256×256 → 128² → 64² → 32² → 16²
↓
AdaptiveAvgPool(4×4) → Flatten = 256×4×4 = 4,096
↓
MLP(4096 → 512)
LeakyReLU(0.2)
↓
μ (mean)
Linear(512→64)
log σ² (variance)
Linear(512→64), clamp[-4, 4]
# Reparameterization Trick
σ = exp(0.5 × log σ²)
ε ~ 𝒩(0, I) ← 표준 정규분포에서 랜덤 샘플링
z₁ = μ + ε × σ ← gradient가 μ, σ를 통해 역전파 가능
z₁ — Latent Vector
shape: (B, 64) · stochastic
z₁이 담고 있는 정보: 같은 설계라도 elevation이 가질 수 있는 "변형"의 특성입니다. μ는 "이 elevation의 평균적 특성", σ는 "얼마나 변형 가능한지"를 나타냅니다.

왜 확률적인가: 하나의 설계에 대해 여러 가능한 elevation이 존재할 수 있으므로, 하나의 고정 벡터가 아닌 분포로 모델링합니다.

추론 시: 이 인코더는 사용되지 않습니다. 대신 z₁ ~ 𝒩(0, I)에서 직접 샘플링합니다. KL loss가 학습 시 z₁의 분포를 𝒩(0, I)에 가깝게 만들어두었기 때문에 이것이 가능합니다.
3

Latent Space Fusion — FiLM으로 z₁과 c 결합

FiLM 적용 ① · 1D 벡터 × 1D 벡터

c(설계 정보)가 z₁(변형 정보)을 element-wise로 변조합니다. "이 설계에서 z₁의 어떤 차원이 중요한지"를 gamma로 스케일링하고, beta로 기본값을 설정합니다.

FiLM Fusion 연산 과정
c
(B, 64)
+
z₁
(B, 64)
↓
γ = Linear(c)
W_γ(64×64) @ c + b_γ → (B, 64)
β = Linear(c)
W_β(64×64) @ c + b_β → (B, 64)
↓
z_fused = γ ⊙ z₁ + β
element-wise: z_fused[i] = γ[i] × z₁[i] + β[i]
# 64차원 중 앞 4개만 추적하는 예시

z₁ = [ 0.50, -1.20, 0.80, 2.00, ... ] ← 랜덤 샘플링된 latent
c = [ 0.30, 0.70, -0.50, 1.10, ... ] ← Design Encoder 출력

# c가 학습된 Linear를 통과하여 γ, β 생성
γ = [ 1.80, 0.05, 2.30, 0.90, ... ] ← 채널별 scale
β = [-0.20, 0.50, 0.00, 1.00, ... ] ← 채널별 shift

# element-wise 적용
z_fused[0] = 1.80 × 0.50 + (-0.20) = 0.70 ← γ=1.8: 증폭됨
z_fused[1] = 0.05 × (-1.20) + 0.50 = 0.44 ← γ≈0: 거의 억제
z_fused[2] = 2.30 × 0.80 + 0.00 = 1.84 ← γ=2.3: 강하게 증폭
z_fused[3] = 0.90 × 2.00 + 1.00 = 2.80 ← γ≈1: 유지 + offset
γ(gamma)의 역할: 설계에 따라 latent의 특정 차원을 증폭(γ>1), 유지(γ≈1), 또는 완전히 억제(γ≈0)합니다. "이 설계에서 z₁의 어떤 변형 축이 중요한가"를 결정합니다.

β(beta)의 역할: 설계 고유의 기본값(offset)입니다. z₁이 0이어도(평균적 변형) β만으로 설계 특성이 반영됩니다.

핵심: γ와 β는 c의 64개 차원 전체가 혼합되어 생성됩니다 (행렬곱). 다른 설계가 들어오면 c가 달라지고, γ와 β도 완전히 달라져서, 같은 z₁이라도 설계에 따라 전혀 다른 z_fused가 생성됩니다.
z_fused

4개 블록이 각각 담당하는 스케일:

Block 1 (8²→16²): 전체적인 elevation의 큰 구조 — 어느 영역이 높고 낮은지
Block 2 (16²→32²): 중간 크기의 구조물 패턴 — 그룹 단위 높이 변화
Block 3 (32²→64²): 개별 컴포넌트 수준의 디테일
Block 4 (64²→128²): 미세한 표면 텍스처, 엣지 디테일

동일한 c가 4개 블록에 들어가지만, 각 블록의 Linear 파라미터가 독립적이므로 같은 c에서도 스케일에 맞는 서로 다른 γ, β를 학습합니다.
5

Loss 계산 및 학습

MSE + KL divergence · β annealing
Loss 구성
x̂ (생성된 elevation)
(B, 1, 256, 256)
vs
x (GT elevation)
(B, 1, 256, 256)
ℒ = MSE(x̂, x) + β × KL( q(z₁|x) ‖ 𝒩(0, I) )
MSE (Reconstruction Loss):
생성된 이미지 x̂와 실제 이미지 x의 픽셀별 차이를 줄입니다. 이 loss가 Design Encoder, Latent Fusion의 Linear, Decoder 전체의 파라미터를 업데이트합니다.
KL Divergence (Regularization):
Elevation Encoder가 만드는 분포 q(z₁|x)를 𝒩(0,I)에 가깝게 만듭니다. 이것이 추론 시 z₁ ~ 𝒩(0,I) 샘플링을 가능하게 합니다. β는 0→0.5로 cyclical annealing하여 posterior collapse를 방지합니다.
Backpropagation 경로:
Loss → Decoder (Conv, FiLM의 γ/β Linear) → z_fused → Latent Fusion (γ/β Linear) → 두 갈래: (1) Elevation Encoder의 μ, σ (2) Design Encoder의 CNN, MLP
모든 파라미터가 동시에 end-to-end로 학습됩니다. FiLM의 Linear들도 이 과정에서 "어떤 γ, β가 좋은 reconstruction을 만드는가"를 학습합니다.

다양성의 유일한 원천은 z₁의 랜덤 샘플링입니다. temperature τ로 다양성을 제어합니다:
τ = 0.5 → z₁이 0 근처에 몰림 → 50개 결과가 비슷 (안전하지만 단조로움)
τ = 1.0 → 표준 정규분포 → 학습 시와 동일한 다양성
τ = 1.5 → 분포가 넓어짐 → 다양하지만 품질 저하 가능 (학습 시 보지 못한 영역)

50개 샘플을 모으면 pixel-wise mean(평균 elevation)과 std(불확실성 맵)를 계산하여 "이 설계에서 elevation이 확실한 부분"과 "변형 가능한 부분"을 분석할 수 있습니다.
